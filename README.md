# الصندوق — تطبيق موبايل لإدارة الحسابات

تطبيق «الصندوق» لإدارة الحسابات حسب الشُّعب (المجموعات) مبني بـ Kivy/KivyMD + SQLite، بدعم كامل للعربية (RTL). تطوير: عمر كريم.

## المزايا
- إضافة الشُّعب وتعديلها وحذفها، مع عرض عدد الأفراد والرصيد الكلي لكل شعبة.
- إضافة الأفراد داخل كل شعبة، ولكل فرد رصيد مستقل.
- **إضافة للكل**: تطبيق مبلغ على جميع أفراد الشعبة دفعة واحدة.
- **تحديد**: اختيار أفراد محددين (مع "تحديد الكل") ثم الإضافة لهم فقط.
- **إضافة / خصم** مبلغ لفرد واحد مع ملاحظة.
- **نقل فرد** من شعبة إلى شعبة أخرى.
- سجل حركات (معاملات) لكل فرد مع التاريخ.
- **ملخّص مالي لكل شعبة**: المبلغ الأصلي (قابل للتعديل) − النثرية − التبرع (قابل للتعديل) = الصافي.
- **النثرية العامة**: مجموع كل الخصومات من جميع الشُّعب في رقم واحد يظهر بجانب «الصندوق» في الأعلى، مع إمكانية **السحب** منه مباشرة (أيقونة 💵−) دون فتح أي شاشة.
- **ثبات**: معالج أخطاء عام يمنع إغلاق التطبيق المفاجئ عند أي خطأ غير متوقع.
- **حقل ماستر كارد** لكل فرد، مع عرض معلوماته (الهاتف والماستر كارد) وأزرار **نسخ** للأرقام.
- **استيراد وتصدير إكسل لكل شعبة**: من داخل الشعبة، استورد أفراداً من ملف `.xlsx`، أو صدّر/نزّل أفراد الشعبة كملف إكسل إلى الهاتف.

## الاستيراد من إكسل
من الشاشة الرئيسية اضغط أيقونة إكسل في الشريط العلوي ثم اختر الملف.
صيغة الملف أعمدة (مع سطر عناوين اختياري): الاسم | الهاتف | الشعبة | ماستر كارد (اختياري).
- إن لم تكن الشعبة موجودة تُنشأ تلقائيًا، والصف بلا شعبة يوضع في «غير مصنّف».
- استخدم الملف الجاهز `قالب_الاستيراد.xlsx` كنقطة بداية.

## الاستيراد والتصدير داخل الشعبة
من شريط أي شعبة:
- **استيراد** (أيقونة الاستيراد): يضيف الأفراد لهذه الشعبة. الأعمدة: الاسم | الهاتف | ماستر كارد (ويدعم أيضًا قالب الأعمدة الأربعة).
- **تصدير** (أيقونة التصدير): يحفظ ملف إكسل بأفراد الشعبة (الاسم | الهاتف | ماستر كارد | الرصيد) في مجلد **التنزيلات** على الهاتف، ويعرض مسار الحفظ مع زر نسخ.

## المظهر
لوحة ألوان احترافية: توولبار كحلي عميق (#1B4965) على خلفية رمادية ناعمة،
بطاقات بيضاء، وأخضر/أحمر مهذّبان للإضافة/الخصم. الألوان معرّفة في أعلى
`main.py` (الثوابت `C_PRIMARY`, `C_POSITIVE` ...) ويمكن تعديلها من مكان واحد.

## التشغيل على سطح المكتب (للتجربة)
```bash
pip install -r requirements.txt
python main.py
```
> ملاحظة: على ويندوز، إذا ظهر النص العربي معكوسًا (عكس مزدوج)،
> افتح `main.py` وغيّر `USE_BIDI = True` إلى `USE_BIDI = False`.

## بناء الـ APK
### الطريقة الأولى: GitHub Actions (موصى بها)
1. ارفع الفولدر كمستودع على GitHub.
2. سيعمل ملف `.github/workflows/build.yml` تلقائيًا عند الـ push،
   أو شغّله يدويًا من تبويب **Actions → Build Android APK → Run workflow**.
3. حمّل ملف الـ APK من قسم **Artifacts** بعد انتهاء البناء.

### الطريقة الثانية: محليًا على لينكس
```bash
pip install buildozer cython
buildozer android debug
# الناتج في مجلد bin/
```

## الخط العربي
الخط المستخدم **IBM Plex Sans Arabic** (احترافي، رخصة OFL). يُسجَّل أي ملف
`.ttf` داخل `assets/fonts/` تلقائيًا (يتعرّف على الوزن العادي والعريض).
لاستبداله بخط آخر (مثل Cairo أو Tajawal)، ضع ملفاته في نفس المجلد.

## قاعدة البيانات
SQLite بثلاث جداول: `divisions`, `members`, `transactions`.
تُحفظ القاعدة داخل بيانات التطبيق (`user_data_dir`) وتُنشأ تلقائيًا في أول تشغيل.
